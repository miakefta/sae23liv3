# CineMatch

## Project presentation

CineMatch is a dynamic web application created for SAE 2.03.

The goal of this project is to help users choose a movie according to their current mood.
Instead of using only classic filters such as actors or genre, the user can search for movies by mood, for example:
- funny
- adrenaline
- relaxing
- thoughtful

The project has two parts:
- an administration interface (backoffice)
- a public interface (frontoffice)

The application is developed in Python and uses an SQLite database.

For the CLI stage, the application currently manages:
- moods
- genres
- movies

A movie is linked to:
- one mood
- one genre

## Python dependencies

This project uses Python 3.

### Standard library modules
- `sqlite3` for the SQLite database
- `pathlib` for file path management
- `dataclasses` for data models

### External library
- `matplotlib` for statistical graphs in CLI mode

### Planned web framework
- `CherryPy` will be used for the web version

## Project files

- `main.py` : CLI administration tool
- `db.py` : database access layer
- `service.py` : business logic layer
- `models.py` : data models
- `seed.py` : insert test data
- `schema.sql` : database schema
- `cinematch.db` : SQLite database file generated by the application

## Installation procedure

1. Copy all project files into the same folder:
   - `main.py`
   - `db.py`
   - `service.py`
   - `models.py`
   - `seed.py`
   - `schema.sql`

2. Make sure Python 3 is installed on your computer.

3. Open a terminal in the project folder.

4. Install `matplotlib` if you want to display graphs:
   ```bash
   pip install matplotlib
   ```

## Configuration files

No environment variables are required for the CLI version.

Important files:
- `schema.sql` contains the SQL commands used to create the database
- `cinematch.db` is the SQLite database file created by the application

If needed, the database file and the schema file can be changed in `service.py`.

Default values:
- database file: `cinematch.db`
- schema file: `schema.sql`

## Database initialization

To initialize the database:

1. Run the CLI program:
   ```bash
   python main.py
   ```

2. Choose option `1`:
   ```text
   Create / reset the database
   ```

3. Insert test data:
   ```bash
   python seed.py
   ```

After that, you can run the CLI again:
```bash
python main.py
```

## CLI usage examples

### Example 1: Create or reset the database

Command:
```bash
python main.py
```

Then choose:
```text
1
```

Expected result:
```text
Base de données créée avec succès.
```

### Example 2: Insert test data

Command:
```bash
python seed.py
```

Expected result:
```text
Données de test insérées avec succès.
```

### Example 3: Search movies

Command:
```bash
python main.py
```

Then choose the search option in the CLI menu and enter for example:
```text
Le titre contient : Inception
Le genre contient :
Année :
ID de l'humeur :
ID du genre :
```

Expected result:
```text
RÉSULTATS : 1 film(s) trouvé(s)
[2] Inception (2010) | Science-fiction | Humeur : Adrénaline | Note : 8.8/10
```

### Example 4: Add a movie

Command:
```bash
python main.py
```

Then go to movie management and choose the option to add a movie.

Example input:
```text
Titre : Avatar
Synopsis : A marine discovers Pandora.
Année : 2009
Durée (minutes) : 162
Note (de 0 à 10) : 7.8
Nom du fichier de l'affiche : avatar.jpg
ID de l'humeur : 2
ID du genre : 3
```

Expected result:
```text
Film ajouté avec succès avec l'ID X.
```
